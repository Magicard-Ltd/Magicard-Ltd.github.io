Package magpy

Magpy is a Python(3) library that can communicate and interact with Magicard's Generation 2 printers.

Installation

To install Magpy, run the following command:

pip3 install magpy

Additional steps for Linux

To communicate with USB printers on Linux you must ensure that the application has permission to access the relevant device files. On Ubuntu this is most easily achieved by either running as root, or by creating a udev rule; for example, create a file /etc/udev/rules.d/98-magicard.rules with the following contents:

SUBSYSTEM=="usb", ATTRS{idVendor}=="0c1f", ATTRS{idProduct}=="0001", MODE:="0666"

Additional steps for Windows

To communicate with USB printers on Windows you must swap the default kernel driver, usbprint for WinUSB. This can be done with Zadig. Note that this will prevent Magicard's Windows driver, and their test tool, from being able to communicate with Magicard printers over USB.

WinUSB can be swapped back to usbprint 'uninstalling' the device (Device Manager -> Universal Serial Bus devices -> Right-click on printer -> Uninstall device -> tick 'Delete the driver software for this device' then click 'Uninstall')

Examples

Monitor printer for errors and display them on screen

from magpy.printer_event_manager import PrinterEventManager
from magpy.printer_event_manager.events import EventId, EventData

# Make a PrinterEventManager object, connected to '1.2.3.4'.
pem = PrinterEventManager("1.2.3.4")

# Event handler.
def on_error_raised(params: EventData) -> None:
    # Create the error code.
    error = params[0].decode() + ":" + params[1].decode()

    # Print the error code.
    print(f"Printer raised {error} error")

# Register the event handler.
pem.register_handler(EventId.ERROR_RAISED, on_error_raised)

# Stream and manage events forever.
pem.events_forever()

Toggle hand-feed mode

from magpy.connection.network_connection import NetworkConnection
from magpy.printer import Printer

# Make a Printer object, connected to '1.2.3.4'.
printer = Printer(NetworkConnection("1.2.3.4"))

# Get the current hand-feed mode value.
hand_fed = printer.get("operation-hand-feed")

# Invert hand-feed and print the result.
hand_fed = printer.set(
    "operation-hand-feed", "false" if hand_fed == "true" else "true"
)

# Print the result.
print(f"Hand feed = {hand_fed}")

Retrieve a printer report/ "query" and save it to disk over USB.

from magpy.connection.usb_connection import UsbConnection
from magpy.printer import Printer

# List all available USB printers.
conns = UsbConnection.get_available_devices()
print("\n".join([str(c) for c in conns]))

# Make a Printer object connected to the first device in the list.
printer = Printer(UsbConnection(conns[0]))

# Download a printer report.
report = printer.get_report()

# Write the report to disk.
with open("report.zip", "wb") as f:
    f.write(report)

Error Status

Name Data type R/RW Description
error String Read-only The printer's current error status.

The error value contains several fields separated by single tab characters (ASCII code 9):

  • The first field is an error code string consisting of the major error code (decimal) followed by a colon character and then the minor error code (decimal).
  • The second field is a brief string description of the error, in the printer's current language. This string may have zero length.
  • The third field is a detailed string description of the error, in the printer's current language. This string may have zero length.
  • The fourth field contains the text for the "Retry" button on the error dialog, in the printer's current language. This string may have zero length, in which case no Retry button should be displayed.
  • The fifth field contains the text for the "Cancel" button on the error dialog, in the printer's current language. This string may have zero length, in which case no Cancel button should be displayed.
  • Any error code with a major code of 0 means there is no error.

Card Position Information

Name Data type R/RW Description
card-position Enumeration Read-only An enumeration indicating the current card position.

The possible values for card-position are:

  • unknown - The card position is unknown.
  • source - There is no card in the print engine.
  • standby - The card is in the standby position.
  • encoder-module - The card is in the optional encoder module.

Note: card-position will be reported as standby when a card is anywhere within the print engine, other than in the encoder module.

wifi-dongle-connected Boolean Read-only True if and only if a WiFi dongle is connected.

1: When reading wifi-connection-status, the failure text is only present when the status is failure. Other states:

  • idle: The service is not in use at all at the moment. It also is not attempting to connect or do anything else.
  • association: Trying to establish a low-level connection to the network.
  • configuration: Trying to retrieve/configure IP settings
  • ready: Successfully connected device. This doesn't mean it has the default route, but basic IP operations will succeed.
  • online: verified connectivity to Internet (ipv4.connman.net can be reached).
  • disconnect: Going to terminate the current connection and will return to the "idle" state.
  • failure: indicates a wrong behavior. It is similar to the "idle" state since the service is not connected.

For all practical purposes ready and online are usually equivalent for the intended connectivity experience. The ready state gives and indication that the network might need a proxy set up to get connected from a company intranet, airport WiFi or similar. In the worst case the network really has limited outside connectivity for one or another reason.
